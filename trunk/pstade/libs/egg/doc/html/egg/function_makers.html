Function Makers

function
function_by_value
function_facade

Function makers build a "baby" function into a Function Object which can be used with boost::lambda::bind and boost::resul_of.

This section uses the following notation.

Valid expression

Semantics

_decltype

An imaginary operator which is the same as decltype of C++0x

_typeof(a)

boost::remove_cv<boost::remove_reference<_decltype(a)>::type>::type

_arg(a)

_typeof(a) const if a is a rvalue; boost::remove_reference<_decltype(a)>::type otherwise.

_ref(a)

static_cast<_typeof(a) const &>(a) if a is a rvalue; a otherwise.

function

Description

function is the "kernel" class template which makes a Bindable Function Object type.

Header
  • <pstade/egg/function.hpp>
Synopsys 
template<class B>
struct function
{
    B b; // exposition only
    B baby() const { return b; }

    // unspecified...
};
Valid expressions

Valid expression

Semantics

function<B>

A Bindable Deferred Callable Object type

function<B>()(a1,...,aN)

boost::add_const<B>::type().call<B::apply<_arg(a1),...,_arg(aN)>::type>(_ref(a1),...,_ref(aN))

function<B>()()

boost::add_const<B>::type().call<B::nullary_result_type>()

Preconditions
  • The corresponding semantics is a valid expression.
Invariants
  • If B is a POD type, function<B> too is a POD type.
Example 

struct baby_second_argument
{
    template<class Myself, class A1, class A2>
    struct apply
    {
        typedef A2 &type;
    };

    template<class Result, class A1, class A2>
    Result call(A1 &a1, A2 &a2) const
    {
        return a2;
    }
};

typedef function<baby_second_argument> op_second_argument;

op_second_argument const second_argument = {{}};

void test()
{
    int i = 2;
    BOOST_CHECK( &(second_argument(1, i)) == &i );
    BOOST_CHECK( second_argument(1, i) == 2 );
}

Don't forget add reference.

Result is A2 &.

A blaced initialization is ok, because op_second_argument is POD.

function_by_value

Description

function turns the rvalue argument into const reference, which is not applicable to a "movable" type like std::auto_ptr. function_by_value simply passes the arguments to the baby by value.

Header
  • <pstade/egg/function_by_value.hpp>
Synopsys 
template<class B>
struct function_by_value
{
    B b; // exposition only
    B baby() const { return b; }

    // unspecified...
};
Valid expressions

Valid expression

Semantics

function_by_value<B>

A Bindable Deferred Callable Object type

function_by_value<B>()(a1,...,aN)

boost::add_const<B>::type().call<B::apply<_typeof(a1),...,_typeof(aN)>::type>(a1,...,aN)

function_by_value<B>()()

boost::add_const<B>::type().call<B::nullary_result_type>()

Preconditions
  • The corresponding semantics is a valid expression.
Invariants
  • If B is a POD type, function_by_value<B> too is a POD type.
Example 

struct baby_value_identity
{
    template<class Myself, class A>
    struct apply
    {
        typedef A type;
    };

    template<class Result, class A>
    Result call(A a) const
    {
        return a;
    }
};

typedef function_by_value<baby_value_identity> op_value_identity;
op_value_identity const value_identity = {{}};

void test()
{
    BOOST_CHECK( value_identity(1) == 1 );
}

A is a "plain" type.

Result is A.

function_facade

Description

Though a type made from function_facade can't be a POD type, it can have non-default constructors unlike function.

Header
  • <pstade/egg/function_facade.hpp>
Valid expressions

Valid expression

Semantics

_typeof(b)

A Bindable Deferred Callable Object type

b(a1,...,aN)

static_cast<_typeof(b) const &>(b).call<_typeof(b)::apply<_arg(a1),...,_arg(aN)>::type>(_ref(a1),...,_ref(aN))

Preconditions
  • b is an object whose type is derived from function_facade<B>.
  • The corresponding semantics is a valid expression.
Example 

template<class T>
struct plus_to
    : function_facade< plus_to<T> >
{
    template<class Myself, class A>
    struct apply
    {
        typedef T type;
    };

    template<class Result, class A>
    Result call(A& a) const
    {
        return m_x + a;
    }

    explicit plus_to(T x)
        : m_x(x)
    {}

private:
    T m_x;
};

template<class T>
plus_to<T> make_plus_to(T x)
{
    return plus_to<T>(x);
}

void test()
{
    BOOST_CHECK( make_plus_to(1)(3) == 4 );
}

plus_to<> is already a full-fledged function.
