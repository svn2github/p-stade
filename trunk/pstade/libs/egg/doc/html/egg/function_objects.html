It is difficult or even impossible to get a Function Object from a function template. Egg no longer considers function templates to be useful, so that it provides "objectified" functions which replace some famous function templates.

Description

Egg provides the Polymorphic Function Object version of <functional>.

Header
  • <pstade/egg/functional.hpp>
Model of
  • ...
Notation
  • do_ is an imaginary operator which applies the corresponding operator.
Valid expressions

Valid expression

Semantics

X_++op<R = _default, _Stg = _default>

A Major Function Object type

X_++op<R, _Stg>()(a1,...,aN)

do_(op, _safe_arg_list(a, _Stg))

op

Static Function Object which behaves as if it were X_++op<>().

Preconditions
  • op is one of the tag names of Boost.Proto.
  • R specifies operator return type. If R is _default, it is deduced by using Boost.Lambdadeduction system.
  • The corresponding semantics is a valid expression.
Example 
BOOST_CHECK(plus(1, 2) == 1 + 2);

Valid expression

Semantics

X_apply<_Stg = _default>

A Major Function Object type

X_apply<_Stg>()(_pfo, a1,...,aN)

_pfo(_safe_arg_list(a, _Stg))

apply

X_apply<>()

Preconditions
  • _safe_arg_list(b, _Stg) is a valid expression, where b is _pfo, a1,...,aN.
  • The corresponding semantics is a valid expression.
Example 

BOOST_CHECK(apply(negate, apply(negate, apply(negate, 10))) == -10);

bll_bind

Description

bll_bind is a Static Function Object which represents boost::lambda::bind.

Header
  • <pstade/egg/bll/bind.hpp>
Model of
  • Static Function Object
Notation
  • ...
Valid expressions

Valid expression

Semantics

bll_bind(f, a1,...,aN)

Polymorphic boost::lambda::bind(f, a1,...,aN)

Preconditions
  • 0 <= N && N <= 9.
  • The corresponding semantics is a valid expression.
Example 

// \x -> (\y -> plus(x, y))
BOOST_CHECK( lazy(bll_bind)(plus, _1, bll::protect(_1))
    (a)(b) == plus(a, b) );

This is currying in Boost.Lambda.
