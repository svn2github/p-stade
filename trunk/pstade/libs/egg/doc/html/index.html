Introduction

“No clever arrangement of bad eggs ever made a good omelet.” -- C.S. Lewis

Unlike other modern languages, it is very difficult to make a function in C++. Egg is a small header-only framework of making functions, and offers higher-order functions.

Requirements

  • Boost C++ Libraries Version 1.34.0 or later (build not required)
  • P-Stade C++ Libraries Version 1.03.9 or later (build not required)

Portability

Egg is known to work on the following platforms:

  • Microsoft Visual C++ .NET Version 7.1 SP1
  • Microsoft Visual C++ 2005 Express Edition SP1
  • GCC 3.4.4
  • GCC 4.1.2

Limitation

  • The maximum arity of any Function Object which Egg offers is 5.
  • Any Function Object which Egg offers is not volatile-aware, meaning that you can't make it volatile-qualified nor pass any volatile object to it.
  • Any Function Object which Egg offers doesn't guarantee to decay the argument passed, meaning that you have to add & to turn a function name into function pointer.

Notation

This document uses the following notation.

Valid expression

Semantics

_decltype

An imaginary operator which is the same as decltype of C++0x.

_typeof(a)

boost::remove_cv<boost::remove_reference<_decltype(a)>::type>::type

_dco

A Deferred Callable Object

_Dco

A Deferred Callable Object type which is neither reference nor cv-qualified.

++

An imaginary operator to concatenate tokens.

Also, assume that every expression is placed after using namespace pstade::egg.
