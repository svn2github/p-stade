Some helper Function Objects are given to fill the gap between Oven and other libraries.

adapted_to/to_base

Description

adapted_to pulls the base iterator from the adapted one.

Header

  • <pstade/oven/adapted_to_base.hpp>

Valid expressions

Valid expression Semantics
oven::adapted_to<B>(a) Calls a.base() until typeof(a.base()) becomes the same as B.
B b = x|to_base; B b = oven::adapted_to<B>(a);

Preconditions

  • B is Copy Constructible.
  • typeof(a) correctly works with boost::is_convertible.
  • There is a (possibly empty) sequence { base(), .., base() } such that typeof(a.base().base()...base()) is B.

Example 

std::string src("cjaigvwzenqhe");
std::string::const_iterator it =
    max_element(
        src
            | filtered(regular(boost::lambda::_1 != 'z'))
            | filtered(regular(boost::lambda::_1 != 'w'))
    )|to_base;

BOOST_CHECK( *it == 'v' );

expression

Description

any_range incurs an overhead for calling a virtual function every iterator operation. The overhead can be removed with the help of Boost.Typeof if your compiler supports the native typeof. expression converts a range into Boost.Typeof compatible one, then removes the overhead if possible.

Header

  • <pstade/oven/expression.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
expression(rng) Returns a Range which can be used with Boost.Typeof macros.

Preconditions

  • boost::range_value<typeof(rng)>::type and boost::range_difference<typeof(rng)>::type are registered to Boost.Typeof.
Note

As far as the two types are registered, expression with Boost.Typeof macros is portable even where native typeof isn't available.

Example 

BOOST_AUTO(factorials,
    expression(counting_from(1)|scanned(1, regular(boost::lambda::_1 * boost::lambda::_2))) );

See also

  • any_range

io.hpp

Description

The header <pstade/oven/io.hpp> lets Ranges Oven returns be Input Streamable and Output Streamable.

Header

  • <pstade/oven/io.hpp>

Valid expressions

Valid expression Semantics
#include <pstade/oven/io.hpp> All the Range Oven defines and returns become a model of Input Streamable and Output Streamable.

Example 

std::string src("abc");
std::stringstream ss;
ss << (src|identities);
BOOST_CHECK( ss.str() == "{a,b,c}" );

nonstop

Description

The Stoppable Generator concept which generation requires is slightly different from the Generator concept. nonstop turns a Generator into a generation conforming one.

Header

  • <pstade/oven/generation.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
nonstop(g) Converts g into a Stoppable Generator.

Preconditions

  • g is a both Generator and Callable Object.

Example 

BOOST_FOREACH (long x, generation(nonstop(&std::rand))|taken(30)) {
    std::cout << "," << x;
}

See also

  • generation

regular

Description

A Boost.Lambda functor is neither Default Constructible nor Assignable. An iterator holding such a functor can't conform to even Input Iterator. regular converts it to comfortable one for iterators.

Header

  • <pstade/oven/regular.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
regular(f) Converts f into a Callable Object which is Default Constructible and Assignable.

Preconditions

  • f is a Boost.Lambda functor or Callable Object.

Example 

BOOST_CHECK( equals(
    std::string("abcdefg")
        | filtered(regular(boost::lambda::_1 != 'c'))
        | filtered(regular(boost::lambda::_1 != 'f')),
    std::string("abdeg")
) );

regular_ref

Description

A Callable Object like a random number generator may not be copyable, while many algorithms require to be. regular_ref, holding a reference to noncopyable Callable Object, behaves as copyable one.

Header

  • <pstade/oven/regular.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
regular_ref(fun) Converts fun into a copyable Callable Object. All the calls are forwarded to fun.

to_upper/to_lower

Description

to_upper and to_lower is the Callable Object version of std::toupper and std::tolower respectively.

Header

  • <pstade/oven/locale.hpp>

Model of

  • Callable Object

Valid expressions

Valid expression Semantics
to_upper(c, l) std::toupper(c, l)
to_upper(c_) to_upper(c_, std::locale())
to_lower(c, l) std::tolower(c, l)
to_lower(c_) to_lower(c_, std::locale())

Preconditions

  • std::to_upper(c, l) is a valid expression.
  • std::to_upper(c_) is a valid expression.

Example 

BOOST_CHECK( equals(
    std::string("ABCDE"),
    std::string("abCde")|transformed(to_upper)
));
Note

to_upper and to_lower is a Callable Object, hence you need not to care whether the base Range is std::string or std::wstring.

See also

  • transformed

unzipped

Description

unzipped reverses zipped.

Header

  • <pstade/oven/unzipped.hpp>

Model of

  • Pipable Object

Valid expressions

Valid expression Semantics
rng|unzipped ({x11,x21,..,xM1} {x12,x22,..xM2} .. {x1N,x2N,xMN})

Preconditions

  • boost::range_value<typeof(rng)>::type is a boost::tuple.
  • There is a xIJ such that {(x11,x12,..x1N),(x21,x22,..,x2N),..,(xM1,xM2,..,xMN)} is rng.

Example 

std::cout <<
    (
        boost::assign::list_of
            (boost::make_tuple(1,2))
            (boost::make_tuple(2,3))
            (boost::make_tuple(3,4))
            | unzipped
    );

// output> ({1,2,3} {2,3,4})

See also

  • zipped

zipped

Description

zipped takes a tuple of ranges and returns a Range of corresponding tuples. If one input Range is short, excess elements of the longer range are discarded.

Header

  • <pstade/oven/zipped.hpp>

Model of

  • Pipable Object

Valid expressions

Valid expression Semantics
u|zipped [T(boost::make_tuple(boost::begin(rng1),..,boost::begin(rngN))), T(boost::make_tuple(boost::end(rng1),..,boost::end(rngN))))

Preconditions

  • u is a boost::tuple such that (rng1 rng2 rngN).
  • T is boost::zip_iterator<typeof(boost::make_tuple(boost::begin(rng1),..,boost::begin(rngN)))> such that T(boost::make_tuple(boost::begin(rng1),..,boost::begin(rngN))) is a valid expression.

Example 

std::cout <<
    (
        boost::make_tuple(
            boost::assign::list_of(1)(2)(3),
            boost::assign::list_of(2)(3)(4)
        )
            | zipped
    );

// output> {(1 2),(2 3),(3 4)}

See also

  • zip_iterator
  • zipped_with
  • unzipped

zipped_with

Description

Though zipped accepts a function taking a boost::tuple as the parameter, it is not so common. zipped_with accepts a "normal" function, then turns it into comfortable one for zipped before passing.

Header

  • <pstade/oven/zipped.hpp>

Valid expressions

Valid expression Semantics
op_make_zipped_with<R> A Callable Object type
op_make_zipped_with<R>()(u, fun) u|zipped|op_make_transformed<R>()(fuse(fun))
op_make_zipped_with<>()(u, cal) u|zipped|op_make_transformed<>()(fuse(cal))
u|zipped_with(cal) op_make_zipped_with<>()(u, cal)

Preconditions

  • u|zipped|op_make_transformed<R>(fuse(fun)) is a valid expressin.
  • u|zipped|transformed(fuse(cal)) is a valid expressin.
  • fuse is a imaginary function which turns a Function Object into the one taking a boost::tuple as the parameter.
