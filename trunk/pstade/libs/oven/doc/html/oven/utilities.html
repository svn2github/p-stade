Some helper Function Objects are given to fill the gap between Oven and other libraries.

adapted_to/to_base

Description

adapted_to pulls the base iterator from the adapted one.

Header

  • <pstade/oven/adapted_to_base.hpp>

Valid expressions

Valid expression Semantics
oven::adapted_to<B>(a) Returns the shortest a.base().base()...base() such that _typeof(a.base().base()...base()) is convertible to B.
B b = x|to_base; B b = oven::adapted_to<B>(a);

Preconditions

  • B is Copy Constructible.
  • _typeof(a) correctly works with boost::is_convertible.
  • There is a (possibly empty) sequence { base(), .., base() } such that _typeof(a.base().base()...base()) is B.

Example 

std::string src("cjaigvwzenqhe");
std::string::const_iterator it =
    max_element(
        src
            | filtered(regular(boost::lambda::_1 != 'z'))
            | filtered(regular(boost::lambda::_1 != 'w'))
    )|to_base;

BOOST_CHECK( *it == 'v' );

io

Description

The header <pstade/oven/io.hpp> lets Oven ranges be Input Streamable and Output Streamable.

Header

  • <pstade/oven/io.hpp>

Effects

All the ranges which Oven defines and returns become Input Streamable and Output Streamable.

Example 

std::string src("abc");
std::stringstream ss;
ss << (src|identities);
BOOST_CHECK( ss.str() == "{a,b,c}" );

nonstop

Description

The Stoppable Generator concept which generation requires is slightly different from the Generator concept. nonstop turns a Generator into a generation conforming one.

Header

  • <pstade/oven/generation.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
nonstop(g) Converts g into a Stoppable Generator.

Preconditions

  • g is a both Generator and Callable Object.

Example 

BOOST_FOREACH (long x, generation(nonstop(&std::rand))|taken(30)) {
    std::cout << "," << x;
}

See also

  • generation

regular

Description

A Boost.Lambda functor is neither Default Constructible nor Assignable. An iterator holding such a functor can't conform to even Input Iterator. regular converts it into comfortable one for iterators.

Header

  • <pstade/oven/regular.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
regular(f) Converts f into a Callable Object which is Default Constructible and Assignable.

Preconditions

  • f is a Boost.Lambda functor or Callable Object.

Example 

BOOST_CHECK( equals(
    std::string("abcdefg")
        | filtered(regular(boost::lambda::_1 != 'c'))
        | filtered(regular(boost::lambda::_1 != 'f')),
    std::string("abdeg")
) );

See also

  • Boost.Lambda
  • filtered
  • transformed

regular_ref

Description

A Callable Object like a random number generator may not be copyable, while many algorithms require to be. regular_ref, holding a reference to noncopyable Callable Object, behaves as copyable one.

Header

  • <pstade/oven/regular.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
regular_ref(_fun) Converts _fun into a copyable Callable Object. All the calls are forwarded to _fun.

to_upper/to_lower

Description

to_upper and to_lower is the Callable Object version of std::toupper and std::tolower respectively.

Header

  • <pstade/oven/locale.hpp>

Model of

  • Callable Object

Valid expressions

Valid expression Semantics
to_upper(c, l) std::toupper(c, l)
to_upper(c_) to_upper(c_, std::locale())
to_lower(c, l) std::tolower(c, l)
to_lower(c_) to_lower(c_, std::locale())

Preconditions

  • std::to_upper(c, l) is a valid expression.
  • std::to_upper(c_, std::locale()) is a valid expression.

Example 

BOOST_CHECK( equals(
    std::string("ABCDE"),
    std::string("abCde")|transformed(to_upper)
));
Note

to_upper and to_lower is a Callable Object, hence you need not to care whether the base Range is std::string or std::wstring.

See also

  • transformed

unzipped

Description

unzipped reverses zipped.

Header

  • <pstade/oven/unzipped.hpp>

Model of

  • Pipable Object

Valid expressions

Valid expression Semantics
_rng|unzipped ({a11,a21,..,aM1} {a12,a22,..,aM2} .. {a1N,a2N,..,aMN})

Preconditions

  • boost::range_value<_typeof(_rng)>::type is a boost::tuple.
  • There is a a such that {(a11 a12 .. a1N),(a21 a22 .. a2N),..,(aM1 aM2 .. aMN)} is _rng.

Example 

std::cout <<
    (
        boost::assign::list_of
            (boost::make_tuple(1,2))
            (boost::make_tuple(2,3))
            (boost::make_tuple(3,4))
            | unzipped
    );

// output> ({1,2,3} {2,3,4})

See also

  • zipped

zipped

Description

zipped takes a tuple of ranges and returns a Range of corresponding tuples. If one input Range is short, excess elements of the longer range are discarded.

Header

  • <pstade/oven/zipped.hpp>

Model of

  • Pipable Object

Valid expressions

Valid expression Semantics
u|zipped [T(boost::make_tuple(_begin(_rng1),..,_begin(_rngN))), T(boost::make_tuple(_end(_rng1),..,_end(_rngN))))

Preconditions

  • u is a boost::tuple such that (_rng1 _rng2 .. _rngN).
  • T is boost::zip_iterator<_typeof(boost::make_tuple(_begin(_rng1),..,_begin(_rngN)))> such that T(boost::make_tuple(_begin(_rng1),..,_begin(_rngN))) is a valid expression.

Example 

std::cout <<
    (
        boost::make_tuple(
            boost::assign::list_of(1)(2)(3),
            boost::assign::list_of(2)(3)(4)
        )
            | zipped
    );

// output> {(1 2),(2 3),(3 4)}

See also

  • zip_iterator
  • zipped_with
  • unzipped

zipped_with

Description

Though zipped accepts a function taking a boost::tuple as the parameter, it is not so common. zipped_with accepts a "normal" function, then turns it into comfortable one for zipped before passing.

Header

  • <pstade/oven/zipped_with.hpp>

Notation

  • fuse is an imaginary function which turns a Function Object into the one taking a boost::tuple as the parameter.

Valid expressions

Valid expression Semantics
op_make_zipped_with<R> A Callable Object type
op_make_zipped_with<R>()(u, _fun) u|zipped|op_make_transformed<R>()(fuse(_fun))
op_make_zipped_with<>()(u, _cal) u|zipped|op_make_transformed<>()(fuse(_cal))
u|zipped_with(_cal) op_make_zipped_with<>()(u, _cal)

Preconditions

  • u|zipped|op_make_transformed<R>(fuse(_fun)) is a valid expressin.
  • u|zipped|transformed(fuse(_cal)) is a valid expressin.
