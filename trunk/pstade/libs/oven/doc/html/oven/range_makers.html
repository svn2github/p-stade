counting

Description

counting makes a boost::counting_iterator range.

Header

  • <pstade/oven/counting.hpp>

Valid expressions

Valid expression Semantics
op_counting<X1,..,XN> A Callable Object type
op_counting<X1,..,XN>()(i, j) [T(i), T(j))
counting(i, j) op_counting<>()(i, j)
counting(i, max_count) op_counting<>()(i, std::numeric_limits<_typeof(i)>::max())

Preconditions

  • 0 <= N && N <= 2
  • T is boost::counting_iterator<_typeof(j), X1,..,XN> such that T(i) and T(j) is a valid expression.

Example 

int ans[] = { 2, 3, 4, 5, 6 };
BOOST_CHECK( equal(counting(2, 7), ans) );

std::vector<int> vec;
BOOST_FOREACH (int i, counting(0, 5)) {
    vec.push_back(i);
}

  • boost::counting_iterator

expression

Description

any_range incurs an overhead for calling a virtual function every iterator operation. The overhead can be removed with the help of Boost.Typeof if your compiler supports the native typeof. expression converts a range into Boost.Typeof compatible one, then removes the overhead if possible.

Header

  • <pstade/oven/expression.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
expression(_rng) [_begin(_rng), _end(_rng)) which can be used with Boost.Typeof macros.

Preconditions

  • boost::range_value<_typeof(_rng)>::type and boost::range_difference<_typeof(_rng)>::type are registered to Boost.Typeof.
Note

As far as the two types are registered, expression with Boost.Typeof macros is portable even where native typeof isn't available.

Example 

BOOST_AUTO(factorials,
    expression(counting_from(1)|scanned(1, regular(boost::lambda::_1 * boost::lambda::_2))) );

See also

  • any_range

generation

Description

generation makes a Single Pass Range from a Stoppable Generator.

Header

  • <pstade/oven/generation.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
generation(g) The longest Single Pass Range {*g(),*g(),*g(),..,*g()} such that g() is not uninitialized.

Preconditions

  • g is a Stoppable Generator.
  • g is Assignable.
  • _typeof(*g()) is Assignable.

Example 

struct rand_generator
{
    typedef boost::optional<long> result_type;

    result_type operator()()
    {
        long result = std::rand();
        if (result % 3 == 0)
            return result_type(); // stop generating.

        return result;
    }
};

void test()
{
    rand_generator X;
    BOOST_FOREACH (long x, generation(X)) {
        std::cout << x << std::endl;
    }
}

See also

  • boost::generator_iterator
  • nonstop
  • regular
  • regular_ref

initial_values

Description

initial_values emulates initializer-lists.

Header

  • <pstade/oven/initial_values.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
initial_values(a1,..,aN) A Random Access Range {a1,..,aN}
Rng rng = initial_values(b1,..,bN); Rng rng = initial_values(b1,..,bN)|copied;

Preconditions

  • 1 <= N && N <= 20
  • _typeof(aK) is convertible to typeof(a1) for all K such that 2 <= K && K <= N.
  • b meets the preconditions of a.
  • Rng rng = initial_values(b1,..,bN)|copied; is a valid expression.
Note

initial_values trades unlimited arity for the lightweight initialization of boost::array.

Example 

int const ans[] = { 1,5,3,6,1,3,7,1,4,2,2 };
std::vector<int> vec = initial_values(1,5,3,6,1,3,7,1,4,2,2);
BOOST_CHECK( equals(vec, ans) );

See also

  • Boost.Array
  • Boost.Assign

iteration

Description

iteration makes an infinite Single Pass Range where the first item is calculated by applying the function on the first argument, the second item by applying the function on the previous result and so on.

Note

Strictly speaking, the range concept doesn't allow an infinite range. So assume here the end iterator is reachable from the begin iterator in the googolplex number of increments.

Header

  • <pstade/oven/iteration.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
iteration(x, _fun) An infinite Single Pass Range {x,_fun(x),_fun(_fun(x)),_fun(_fun(_fun(x))),...}

Preconditions

  • _fun(_fun(x)) is a valid expression.
  • _fun is Assignable.

Example 

int answer[] = { 1,2,4,8,16 };
BOOST_CHECK( equals(answer,
    iteration(1, regular(boost::lambda::_1 * 2))|taken(5)
) );

See also

  • taken

reading

Description

reading makes a Single Pass Range from std::cout etc.

Header

  • <pstade/oven/reading.hpp>

Valid expressions

Valid expression Semantics
op_reading<V, D> A Callable Object type
op_reading<V, D>()(s) [T(s), T())
op_reading<V> op_reading<V, std::ptrdiff_t>

Notice that reading<V>(s) is qualified with oven::.

  • T is std::istream_iterator<V, _typeof(s)::char_type, _typeof(s)::traits_type, D> suct that T(s) is a valid expression.

recursion

Description

recursion, collaborating with any_range, creates a recursive range.

Header

  • <pstade/oven/recursion.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
recursion(_fwdrng) An infinite range up to Bidirectional Range [_begin(_fwdrng), _end(_fwdrng))

Preconditions

  • _fwdrng is an any_range.
  • Every memoized in the base range takes its own memo_table.

Example 

typedef any_range<int const&, boost::forward_traversal_tag> range_t;
range_t fibs;
memo_table tb;
int const start[] = { 1, 1 };
fibs =
    start
        | transformed(pstade::as_value)
        | jointed(
            boost::make_tuple(recursion(fibs), recursion(fibs)|dropped(1))
                | zipped_with(regular(boost::lambda::_1 + boost::lambda::_2))
            )
        | memoized(tb)
;

std::cout << (fibs|taken(howMany));
Note

In a recursive range, memoized must take a named memo_table object. A recursive range tends to be inefficient without memoization.

See also

  • any_range
  • memoized

shared

Description

shared makes a range from a pointer to a heap allocated range, and the iterators manage the allocation.

Header

  • <pstade/oven/shared.hpp>

Model of

  • Pipable Object

Valid expressions

Valid expression Semantics
p|shared [T(_begin(*p), sp), T(_end(*p), sp))

Preconditions

  • sp is a boost::shared_ptr<_typeof(*p)>(p) such that sp(p) is a valid expression.
  • T is a shared_range_iterator<_typeof(*p)> such that T(_begin(*p), sp) is a valid expression.
  • shared_range_iterator is an imaginary iterator which works like boost::shared_container_iterator but uses a range instead of a container.

Example 

BOOST_FOREACH (char ch, std::string("dangling")|identities) {
    // will crash; 'std::string' object doesn't exist anymore. 
    std::cout << ch;
}

BOOST_FOREACH (char ch, new std::string("ok")|shared|identities) {
    // works fine.
    std::cout << ch;
}

See also

  • shared_container_iterator
  • shared_single

single

Description

single makes a Random Access Range which delivers a range presentation of one object.

Header

  • <pstade/oven/single.hpp>

Model of

  • Auxiliary Interface

Valid expressions

Valid expression Semantics
single(x) [&x, &x + 1)
Note

As the semantics implies, the iterators are valid as long as x is not destructed. If you want the iterators to manage the base object lifetime, use shared_single.

Example 

BOOST_CHECK( equals('a'|single, std::string("a")) );
