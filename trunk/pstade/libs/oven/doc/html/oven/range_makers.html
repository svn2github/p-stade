as_array

Description

Some versions of Boost.Range regard char array as literal, which as_array works around.

Header

  • <pstade/oven/as_array.hpp>

Model of

  • Auxiliary Interface

Valid expressions

Valid expression Semantics
as_array(a) as_array(a) of Proposal for new string algorithms in TR2.

Preconditions

  • a is an array.

as_c_str

Description

as_c_str makes a Random Access Range from null-terminated c-style string.

Header

  • <pstade/oven/as_c_str.hpp>

Model of

  • Auxiliary Interface

Valid expressions

Valid expression Semantics
as_c_str(s) [s, s + strlen(s))
as_c_str(rng) [boost::begin(rng), std::find(boost::begin(rng), boost::end(rng), 0))

Preconditions

  • typeof(s) is convertible to char * or char const *.
  • typeof(rng) is not convertible to char * nor char const *.

as_literal

Description

as_literal makes a Random Access Range from character array.

Header

  • <pstade/oven/as_literal.hpp>

Model of

  • Auxiliary Interface

Valid expressions

Valid expression Semantics
as_literal(a) [&a[0], &a[0]+sz-1), where sz is the size of array a.
as_literal(x) x

Preconditions

Note

as_literal doesn't use strlen, while Proposal for new string algorithms in TR2 does.

Valid expression Semantics

}

initial_values

Description

initial_values emulates initializer-lists.

Header

  • <pstade/oven/initial_values.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
initial_values(a1,..,aN) boost::assign::cref_list_of<N>(a1)..(aN)

Preconditions

  • 1 <= N && N <= 20
  • boost::assign::cref_list_of<N>(a1)..(aN) is a valid expression.
Note

initial_values trades unlimited arity for the lightweight initialization of boost::array.

Example 

int const ans[] = { 1,5,3,6,1,3,7,1,4,2,2 };
std::vector<int> vec = initializers(1,5,3,6,1,3,7,1,4,2,2);
BOOST_CHECK( equals(vec, ans) );

iteration

Description

iteration makes an infinite Single Pass Range where the first item is calculated by applying the function on the first argument, the second item by applying the function on the previous result and so on.

Note

Strictly speaking, the Range concept doesn't allow an infinite range. So assume here the end iterator is reachable from the begin iterator in the googolplex number of increments.

Header

  • <pstade/oven/iteration.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
iteration(x, fun) Returns an infinite Single Pass Range which behaves as if it were { x, fun(x), fun(fun(x)), fun(fun(fun(x))), ... }

Preconditions

  • fun(x) is a valid expression.

Example 

int answer[] = { 1,2,4,8,16 };
BOOST_CHECK( equals(answer,
    iteration(1, regular(boost::lambda::_1 * 2))|taken(5)
) );

recursion

Description

recursion, collaborating with any_range, creates a recursive range.

Note

In a recursive range, memoized must take a named memo_table object. A recursive range tends to be inefficient without memoization.

Header

  • <pstade/oven/recursion.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
recursion(fwdrng) Returns an infinite Range up to Bidirectional Range.

Preconditions

  • fwdrng is an any_range.

Example 

typedef any_range<int const&, boost::forward_traversal_tag> range_t;
range_t fibs;
memo_table tb;
int const start[] = { 1, 1 };
fibs =
    start
        | transformed(pstade::as_value)
        | jointed(
            boost::make_tuple(recursion(fibs), recursion(fibs)|dropped(1))
                | zipped_with(regular(boost::lambda::_1 + boost::lambda::_2))
            )
        | memoized(tb)
;

std::cout << (fibs|taken(howMany));

See also

  • any_range

shared

Description

shared makes a Range from a pointer to a heap allocated Range, and the iterators manage the allocation.

Header

  • <pstade/oven/shared.hpp>

Model of

  • Pipable Object

Valid expressions

Valid expression Semantics
p|shared [T(), T())

Preconditions

  • boost::shared_ptr is constructible from p.

Example 

BOOST_FOREACH (char ch, std::string("dangling")|identities) {
    // will crash; 'std::string' object doesn't exist anymore. 
    std::cout << ch;
}

BOOST_FOREACH (char ch, new std::string("ok")|shared|identities) {
    // works fine.
    std::cout << ch;
}

See also

  • shared_container_iterator

stream_input

Description

stream_input makes a Single Pass Range from std::cout etc.

Header

  • <pstade/oven/stream_input.hpp>

Valid expressions

Valid expression Semantics
op_stream_input<V, D> A Callable Object type
op_stream_input<V> op_stream_input<V, std::ptrdiff_t>
op_stream_input<V, D>()(s) [T(s), T())
oven::stream_input<V>(s) op_stream_input<V>()(s)
Note

Notice that stream_input<V>(s) is qualified with oven::.

Preconditions

  • T is std::istream_iterator<V, typeof(s)::char_type, typeof(s)::traits_type, D> suct that T(s) is a valid expression.

Example 

std::string src("hello,stream_input!");

std::stringstream ss;
ss << src;

std::string result;
copy(oven::stream_input<char>(ss), std::back_inserter(result));

BOOST_CHECK( equals(result, src) );

See also

  • istream_iterator
