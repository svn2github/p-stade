Oven provides range-based algorithms using a Range instead of two iterators.

STL Algorithms

Description

algorithm.hpp is considered as range-based <algorithm>.

Header

  • <pstade/oven/algorithm.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
algo1(_rng, x1,..,xN) std::algo1(_begin(_rng), _end(_rng), x1,..,xN)
algo2(_rng1, _rng2, x1,..,xN) std::algo2(_begin(_rng1), _end(_rng1), _begin(_rng2), _end(_rng2), x1,..,xN)
algoM(_rng, _fun, x1,..,xN) std::algoM(_begin(_rng), _fun(_rng), _end(_rng), x1,..,xN)

Preconditions

  • algo1 and algo2 is one of the function names defined in <algorithm> except for...
    • fill_n, generate_n, rotate, rotate_copy, partial_sort, nth_element and inplace_merge.
  • std::algo1(_begin(_rng), _end(_rng), x1,..,xN) is a valid exression.
  • std::algo2(_begin(_rng1), _end(_rng1), _begin(_rng2), _end(_rng2), x1,..,xN) is a valid expression.
  • algoM is one of the following names...
    • rotate, rotate_copy, partial_sort, nth_element and inplace_merge.
  • std::algoM(_begin(_rng), m, _end(_rng), x1,..,xN) is a valid expression.
  • _fun(_rng) is a valid expression.
  • _typeof(_fun(_rng)) is _iter_of<_typeof(_rng)>::type.
Note

fill_n and generate_n are not considered as range algorithm, so they are excluded.

Example 

std::string str;

// iterator-based
str = "gfedcba";
std::sort(str.begin(), str.end());
BOOST_CHECK( str == "abcdefg" );

// Oven range-based
str = "gfedcba";
sort(str);
BOOST_CHECK( str == "abcdefg" );

STL Numerics

Description

numeric.hpp is considered as range-based <numeric>.

Header

  • <pstade/oven/numeric.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
algo(_rng, x1,..,xN) std::algo(_begin(_rng), _end(_rng), x1,..,xN)

Preconditions

  • algo is one of the function names defined in <numeric>.
  • std::algo(_begin(_rng), _end(_rng), x1,..,xN) is a valid expression.
Note

Unfortunately, Oven STL Algorithms and Numerics are not traversal-aware. A Random Access Range whose iterator is not Lvalue Iterator is regarded as an Input Iterator range.

at

Header

  • <pstade/oven/at.hpp>

Model of

  • Auxiliary Interface

Valid expressions

Valid expression Semantics
at(_rndrng, n) *(_begin(_rndrng) + n)

Preconditions

  • _typeof(n) is convertible to boost::range_difference<_typeof(_rndrng)>::type.
  • 0 <= n && n < distance(_rndrng)
  • Destruction of an iterator doesn't invalidate references previously obtained from that iterator.
Note

you must use value_at if an iterator invalidates references after the destruction.

Example 

std::string str("f12344513215b");
BOOST_CHECK( at(str, 0) == 'f' );
BOOST_CHECK( (str|at(4)) == '4' );

See also

  • value_at

begin/end

Description

begin and end is the Auxiliary Interface version of _begin and _end respectively.

Header

  • <pstade/oven/begin_end.hpp>

Model of

  • Auxiliary Interface

Valid expressions

Valid expression Semantics
begin(_rng) _begin(_rng)
end(_rng) _end(_rng)

Example 

std::string src("abcDefg");   
oven::copy(src|reversed|transformed(to_upper), src|reversed|begin);
BOOST_CHECK( oven::equals(src, std::string("ABCDEFG")) );

copied

Description

copied automagically calls boost::copy_range without specifying the return type.

Header

  • <pstade/oven/copy_range.hpp>

Valid expressions

Valid expression Semantics
Rng1 _rng1 = _rng2|copied; Rng1 _rng1 = boost::copy_range<Rng1>(_rng2);

Preconditions

  • Rng1 _rng1 = boost::copy_range<Rng1>(_rng2); is a valid expression.
  • copied isn't placed in a function default argument list.
Note

The valid expression implies you can place copied everywhere it is used in copy-initialization, e.g. return statement. The second precondition comes from a GCC bug. Notice that Rng1 _rng1(_rng2|copied); is not listed in valid expressions.

Example 

std::string _rng("abcdefg");
std::vector<char> vec = _rng|copied;
BOOST_CHECK( equals(vec, _rng) );

distance

Description

distance is a range-based std::distance but traversal-aware.

Header

  • <pstade/oven/distance.hpp>

Model of

  • Auxiliary Interface

Valid expressions

Valid expression Semantics
distance(_rng) std::distance(_begin(_rng), _end(_rng))

equals

Description

equals is a range-based std::equal but sensitive to the distances.

Header

  • <pstade/oven/equals.hpp>

Model of

  • Callable Object

Valid expressions

Valid expression Semantics
equals(_rng1, _rng2, _prd) distance(_rng1) == distance(_rng2) ? std::equal(_begin(_rng1), _end(_rng1), _begin(_rng2), _prd) : false
equals(_rng1, _rng2) equals(_rng1, _rng2, equal_to)

Preconditions

  • equal_to = (boost::lambda::_1 == boost::lambda::_2)

front/back

Description

Header

  • <pstade/oven/front_back.hpp>

Model of

  • Auxiliary Interface

Valid expressions

Valid expression Semantics
front(_rng) *_begin(_rng)
back(_bidrng) *boost::prior(_bidrng)

Preconditions

  • !boost::empty(_rng)
  • !boost::empty(_bidrng)
  • Destruction of an iterator doesn't invalidate references previously obtained from that iterator.

See also

  • value_front/value_back

value_at

Description

Header

  • <pstade/oven/at.hpp>

Model of

  • Auxiliary Interface

Valid expressions

Valid expression Semantics
value_at(_rndrng, n) boost::range_value<_typeof(_rndrng)>::type(*(_begin(_rndrng) + n))

Preconditions

  • boost::range_value<_typeof(_rndrng)>::type is Copy Constructible.
  • _typeof(n) is convertible to boost::range_difference<_typeof(_rndrng)>::type.
  • 0 <= n && n < distance(_rndrng)

See also

  • at

value_front/value_back

Description

Header

  • <pstade/oven/front_back.hpp>

Model of

  • Auxiliary Interface

Valid expressions

Valid expression Semantics
value_front(_rng) boost::range_value<_typeof(_rng)>::type(*_begin(_rng))
value_back(_bidrng) boost::range_value<_typeof(_bidrng)>::type(*boost::prior(_bidrng))

Preconditions

  • boost::range_value<_typeof(_rng)>::type is Copy Constructible.
  • boost::range_value<_typeof(_bidrng)>::type is Copy Constructible.
  • !boost::empty(_rng)
  • !boost::empty(_bidrng)

See also

  • front/back
