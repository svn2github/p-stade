Bindable Object

Description

A Bindable Object is a Function Object which can be passed to boost::lambda::bind.

Refinement of

  • Function Object

Requirements

In addition to the requirements defined in Function Object, for any Bindable Object b the following must be met:

  • There is a N such that boost::lambda::bind(b, boost::lambda::_1,..,boost::lambda::_N) is a valid expression.

Models

See also

  • Boost.Lambda

Operatable Object

Description

An Operatable Object is a Deferred Callable Object which can specify the type of itself using prefix op_.

Refinement of

  • Deferred Callable Object
  • Bindable Object

Requirements

In addition to the requirements defined in Deferred Callable Object and Bindable Object, for any Operatable Object f the following must be met:

  • op_f is _typeof(f).

Models

  • make_filtered
  • generation
Caution

Because of a compiler bug, boost::result_of of Boost version 1.34 or below is utterly broken on msvc-7.1 and 8.0. pstade::result_of  in <pstade/result_of.hpp> can be used for the workaround.

Pipable Object

Description

A Pipable Object supports a pipe-like syntax using operator| and has the associated Operatable Object using prefix make_.

Refinement of

  • Deferred Callable Object

Requirements

For any Pipable Object p the following must be met:

  • x|p(x1,..,xN) is a valid expression if and only if make_p(x, x1,..xN) is a valid expression.
  • x|p(x1,..,xN) and make_p(x, x1,..xN) have the same effects.
  • make_p is an Operatable Object.
  • If N is 0, x|p and make_p(x) are valid expressions which have the same effects.

Models

  • filtered
  • transformed

Auxiliary Interface

Description

An Auxiliary Interface is an Operatable Object which supports also a pipe-like syntax used as if it were a member function.

Refinement of

  • Operatable Object

Requirements

In addition to the requirements defined in Operatable Object, for any Auxiliary Interface i the following must be met:

  • x|i(x1,..,xN) is a valid expression if and only if i(x, x1,..,xN) is a valid expression.
  • x|i(x1,..xN) and i(x, x1,..,xN) have the same effects.
  • If N is 0, x|i() and x|i are valid expressions which have the same effects.

Models

  • as_array
  • begin/end

See also

  • Auxiliary class interfaces

Stoppable Generator

Description

A Stoppable Generator is a Deferred Callable Object which can specify the stop of generating using Boost.Optional.

Refinement of

  • Deferred Callable Object

Requirements

In addition to the requirements defined in Deferred Callable Object, for any Stoppable Generator g the following must be met:

  • _typeof(g(..)) is a boost::optional.
  • An uninitialized result is considered as the stop of generating.

Models

  • Any Deferred Callable Object which nonstop returns.

Constant Range

Description

A Constant Range is a Range whose elements are not modifiable.

Refinement of

  • Single Pass Range

Requirements

In addition to the requirements defined in Single Pass Range, for any Constant Range r the following must be met:

  • boost::implicit_cast<_value_of<_typeof(r)>::type&>(*_begin(_r)) is ill-formed.
  • r is neither Writable nor Swappable.

Models

  • std::list<int> const
  • int const[10]

Contiguous Range

Description

A Contiguous Range is a Random Access Range whose elements are stored contiguously.

Refinement of

  • Random Access Range
  • Lvalue Range

Requirements

In addition to the requirements defined in Random Access Range, for any Contiguous Range r the following must be met:

  • &*(_begin(r) + n) == &*(_begin(r)) + n for all n such that 0 <= n && n < _end(r) - _begin(r).

Models

  • std::vector<int>
  • boost::array<char, 50>
