STL Algorithms

Description

Oven provides most of all the STL algorithms as range-based functions which take a range instead of two iterators.

Header

  • <pstade/oven/algorithm.hpp>

Model of

  • Operatable Object

Valid expressions

Valid expression Semantics
algo(rng, a0,..,aN) std::algo(boost::begin(rng), boost::end(rng), a0,..,aN)

Preconditions

  • algo is one of the function names defined in <algorithm> except for...
    • fill_n, generate_n, rotate, rotate_copy, partial_sort, nth_element and inplace_merge.
  • std::algo(boost::begin(rng), boost::end(rng), a0,..,aN) is a valid exression.
Note

Unfortunately, Oven STL algorithms are not traversal-aware, that is to say, a Random Access Range whose iterator is not Lvalue Iterator is regarded as an Input Iterator range.

Example 

std::string str;

// iterator-based
str = "gfedcba";
std::sort(str.begin(), str.end());
BOOST_CHECK( str == "abcdefg" );

// Oven range-based
str = "gfedcba";
sort(str);
BOOST_CHECK( str == "abcdefg" );

at

Header

  • <pstade/oven/at.hpp>

Model of

  • Auxiliary Interface

Valid expressions

Valid expression Semantics
at(rndRng, n) boost::begin(rndRng)[n]

Preconditions

  • 0 <= n && n < distance(rndRng)
  • Destruction of an iterator doesn't invalidate references previously obtained from that iterator.
Note

you must use value_at if an iterator invalidates references after the destruction.

Valid expression Semantics

Example 

std::string src("abcDefg");   
oven::copy(src|reversed|transformed(to_upper), src|reversed|begin);
BOOST_CHECK( oven::equals(src, std::string("ABCDEFG")) );

copied

Header

  • <pstade/oven/copy_range.hpp>

Valid expressions

Valid expression Semantics
Rng1 rng1 = rng2|copied; Rng1 rng1 = boost::copy_range<Rng1>(rng2);

Preconditions

  • Rng1 rng1 = boost::copy_range<Rng1>(rng2); is a valid expression.
  • copied isn't placed in a function default argument list.
Note

The valid expression implies you can place copied everywhere it is used in copy-initialization, e.g. return statement. The second precondition comes from a GCC bug. Notice that Rng1 rng1(rng2|copied); is not listed in valid expressions.
