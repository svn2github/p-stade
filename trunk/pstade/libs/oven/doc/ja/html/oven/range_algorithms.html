Oven は二つのイテレータを取るアルゴリズムの代わりに、range-base のアルゴリズムを提供する。

STL Algorithms

Description

algorithm.hpp は range-base の <algorithm> を提供する。

Header
  • <pstade/oven/algorithm.hpp>
Model of
  • Static Function Object
Valid expressions

Valid expression

Semantics

f1(_rng, x1,...,xN)

std::f1(_begin(_rng), _end(_rng), x1,...,xN)

f2(_rng1, _rng2, x1,...,xN)

std::f2(_begin(_rng1), _end(_rng1), _begin(_rng2), _end(_rng2), x1,...,xN)

f3(_rng, _fun, x1,...,xN)

std::f3(_begin(_rng), _fun(_rng), _end(_rng), x1,...,xN)

Preconditions

  • f1、f2は <algorithm> にある関数のいずれかに該当する。ただし、以下のものを除く。

    • fill_n, generate_n, rotate, rotate_copy, partial_sort, nth_element および inplace_merge.

  • f3 は以下のいずれかに該当する。

    • rotate, rotate_copy, partial_sort, nth_element および inplace_merge.
  • std::f1(_begin(_rng), _end(_rng), x1,...,xN) は有効な式である。
  • std::f2(_begin(_rng1), _end(_rng1), _begin(_rng2), _end(_rng2), x1,...,xN) は有効な式である。
  • std::f3(_begin(_rng), m, _end(_rng), x1,...,xN) は有効な式である。
  • _fun(_rng) は有効な式である。
  • _typeof(_fun(_rng)) は _iter_of<_typeof(_rng)>::type である。
Note

fill_n および generate_n は range algorithm とは見做されないため除外した。

Example 
std::string str;

// iterator-based
str = "gfedcba";
std::sort(str.begin(), str.end());
BOOST_CHECK( str == "abcdefg" );

// Oven range-based
str = "gfedcba";
sort(str);
BOOST_CHECK( str == "abcdefg" );

at

Description

at は range の始点からN個目の要素を返す。

Header
  • <pstade/oven/at.hpp>
Model of
  • Ambi Static Function Object
Valid expressions

Valid expression

Semantics

at(_rndrng, n)

*(_begin(_rndrng) + n)

Preconditions
  • _rndrng は Readable もしくは Lvalue である。
  • boost::range_reference<_typeof(_rndrng)>::type が reference type の場合、_rndrng は Lvalue Range である。
  • _typeof(n) は boost::range_difference<_typeof(_rndrng)>::type と交換可能である。
  • 0 <= n && n < distance(_rndrng)
Note

 二つ目の前提条件を満たさない場合、value_at を利用しなければならない。

Example 
std::string str("f12344513215b");
BOOST_CHECK( at(str, 0) == 'f' );
BOOST_CHECK( (str|at(4)) == '4' );
See also
  • value_at

copied

Description

copied は型指定なしで自動的に対象 range の range-constructor を呼ぶ。

Model of
  • Pipable Adaptor
Header
  • <pstade/oven/copied.hpp>
Notation
  • V1 is _value_of<_typeof(_rng1)>::type.
  • to_v1 is regular(boost::lambda::ll_static_cast<V1>(boost::lambda::_1)).
Valid expressions

Valid expression

Semantics

Rng1 _rng1 = _rng|copied;

Rng1 _rng1(_begin(r), _end(r)), where r = _rng|transformed(to_v1).

Rng2 _rng2(_rng0|copied);

Rng2 _rng2 = _rng0|copied;

Preconditions
  • _rng1 は Sequence である。
  • type X_ は boost::is_convertible<X_, Rng1>::value == false となるような型である。
  • _value_of<_typeof(_rng)>::type は V1 と交換可能である。
  • Rng2 は Rng2 _rng2 = _rng0|copied; が有効な式となるような aggregate type である。
  • copied は関数のデフォルト引数リストに配置されない。
Note

最初の例は return statement のように、copy-initialization が利用可能ならば copied に置き替えられる事を意味している。最後の制限はGCCのバグによるものである。Rng1 _rng1(_rng2|copied); はいつも有効な式であるとは限らない点に注意せよ。

Example 
std::string rng("abcdefg");
std::vector<char> vec = rng|copied;
BOOST_CHECK( equals(vec, rng) );
See also
  • transformed

equals

Description

equals は厳密な範囲チェックを行なう range-base の std::equal である。

Header
  • <pstade/oven/equals.hpp>
Model of
  • Static Function Object
Notation
  • eq は (boost::lambda::_1 == boost::lambda::_2)である。
Valid expressions

Valid expression

Semantics

equals(_rng1, _rng2, _prd)

distance(_rng1) == distance(_rng2) ? std::equal(_begin(_rng1), _end(_rng1), _begin(_rng2), _prd) : false

equals(_rng1, _rng2)

equals(_rng1, _rng2, eq)

Preconditions
  • _rng1 と _rng2 は Readable または Lvalue である。
