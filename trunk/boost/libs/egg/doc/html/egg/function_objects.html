It is difficult or even impossible to get a Function Object from a function template. Egg no longer considers function templates to be useful, so that it provides "objectified" functions which replace some famous function templates.

always

Description

always returns a Function Object always returning the same object.

Header
  • <boost/egg/always.hpp>
Model of
  • Static Function Object
Notation
  • b is an object of type which is not boost::reference_wrapper<>.
  • w is an object of boost::reference_wrapper<>.
Valid expressions

Valid expression

Semantics

always(x)

A Major Function Object type

always(b)(a1,...,aN)

boost::implicit_cast<_typeof(b)>(b)

always(w)(a1,...,aN)

w.get()

Preconditions
  • 0 <= N && N <= BOOST_EGG_MAX_LINEAR_ARITY.
  • The corresponding semantics is a valid expression.

bll_bind

Description

bll_bind is a Static Function Object which represents boost::lambda::bind.

Header
  • <boost/egg/bll/bind.hpp>
Model of
  • Static Function Object
Notation
  • ...
Valid expressions

Valid expression

Semantics

bll_bind(f, a1,...,aN)

Polymorphic boost::lambda::bind(f, a1,...,aN)

Preconditions
  • 0 <= N && N <= 9.
  • The corresponding semantics is a valid expression.
Example 

// \x -> (\y -> plus(x, y))
BOOST_CHECK( lazy(bll_bind)(plus, _1, bll::protect(_1))
    (a)(b) == plus(a, b) );

This is currying in Boost.Lambda.
