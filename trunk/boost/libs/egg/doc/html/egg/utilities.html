expr

Description

"typeof" and "type-erasure" are not conflicting forces. expr performs type-erasure when native typeof isn't available. So that, type registrations into Boost.Typeof can be minimized.

Header
  • <boost/egg/expr.hpp>
Model of
  • ...
Notation
  • ...
Valid expressions

Valid expression

Semantics

egg::expr<Sig>(_pfo)

X_mono<Sig>()(_pfo) which can be used with Boost.Typeof.

Preconditions
  • Except for boost::use_default, types in Sig is registered to Boost.Typeof.
  • The corresponding semantics is a valid expression.
Example 

void egg_example()
{
    BOOST_AUTO(f,
        egg::expr<int(int, int)>(unfuse(fuse(bll::_1 + bll::_2))) );

    BOOST_CHECK( f(1,2) == 1+2 );
}

infix

Description

Egg provides FC++ infix operator syntax.

Header
  • <boost/egg/infix.hpp>
Model of
  • ...
Notation
  • ...
Valid expressions

Valid expression

Semantics

a1 ^_pfo^ a2

_pfo(_fwd_arg_list([a1], by_perfect)++_fwd_arg_list([a2], by_perfect))

Preconditions
  • using namespace infix; is placed before the valid expression.
  • operator^ is not overloaded with _typeof(a1).
  • The corresponding semantics is a valid expression.
Invariants
  • ^_pfo^ behaves as if it had left to right associativity.
Example 

void egg_example()
{
    using namespace infix;

    std::plus<int> plus;
    BOOST_CHECK( (1 ^plus^ 2) == 3 );
}
