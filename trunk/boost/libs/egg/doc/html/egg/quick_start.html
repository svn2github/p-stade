Installing Egg

Getting Egg

You can get egg.zip from Boost.Vault/FunctionObjects. Also, if you have a svn client, Egg svn repository is available using the following command: 

svn co https://p-stade.svn.sourceforge.net/svnroot/p-stade/trunk/boost/ boost_egg
Building Egg

Egg is a header-only library. It is unneeded to build or link to any lib files.

Requirements

Egg requires Boost C++ Libraries Version 1.34.1 or later. You don't need to build any Boost libraries.

Portability

Egg is known to work on the following platforms:

  • Microsoft Visual C++ Version 7.1 or later
  • GCC 3.4.4 or later

Using Egg

Hello Builder

In short, Egg is a library for building functions. Is C++ a language which requires library for building functions? The answer might be yes if you want to build Polymorphic Function Object type. Below is the minimal example building such a function: 

#include <boost/egg/function.hpp>
#include <boost/egg/result_of.hpp>
#include <iostream>

using namespace boost::egg;

struct little_identity 
{
    template<class Me, class T>
    struct apply
    {
        typedef T &type; 
    };

    template<class Re, class T>
    Re call(T &t) const
    {
        return t;
    }
};

typedef function<little_identity> T_identity; 
T_identity const identity = {{}};

int main()
{
    int i = 10;
    result_of_<T_identity(int &)>::type r = identity(i); 
    std::cout << r; 

    return 0;
}

Conforms Little Function.

The return type is depend on argument type. In other words, it is Polymorphic.

function is one of the Function Builders.

identity can be used with Boost.ResultOf.

Prints 10.

Egg defines the concept, Little Function, from which Egg framework grows Major Function Object type. You can choose your favorite builder from Function Builders, then pass Little Function to it. In the example above, function is used, which is the most primitive and lightweight builder.

Hello Adaptor

Egg provides Function Adaptors also known as higher-order functions: 

#include <boost/egg/lazy.hpp>
#include <boost/lambda/core.hpp>
#include <iostream>

using namespace boost::egg;

struct T_plus
{
    typedef int result_type;

    result_type operator()(int x, int y) const 
    {
        return x + y;
    }
};

result_of_lazy<T_plus>::type
    const Plus = BOOST_EGG_LAZY({}); 

int main()
{
    namespace bll = boost::lambda;
    int i10 = 10;
    std::cout << Plus(bll::_1, 20)(i10); 

    return 0;
}

Don't forget const-qualifer.

Making T_plus lazy, which can be used with Boost.Lambda placeholders.

Prints 30.

Function Adaptors provides several "views" into functions. You can adapt a function into lazy, pipable, or tagged function etc on demand.

Reading Doc

The organization of Egg is "flat". Egg can be regarded as federation of Function Builders and Function Adaptors. Each component is usually independent from each other, so that you can read the sections with any order.

Notation

Notation introduces several keywords for documentation. A word which begins with __ is a keyword. When you forget its semantics, just click the keyword.

Concepts

Concepts describes the concepts used in Egg. By the design rationale, Concepts doesn't introduce any new "traits" something like is_bindable<> etc. Those just define the valid expressions and semantics. The most important concepts are Little Function and Major Function Object. Other conepts could be read on demand.

Forwarding Strategies

Forwarding Strategies defines strategy tags for argument forwarding. Most of all Egg components take a parameter for them. Fortunately, the default behavior, by_perfect, works so well in most cases that you could skip to read Forwarding Strategies for now.

Function Builders

Function Builders build Major Function Object type. As mentioned above, function is the most primitive builder. First, it might be preferable to understand function with Little Function concept. Second, I recommend poly, which is a cool builder originally written by David Abrahams.

Function Adaptors

Function Adaptors are higher-order functions which take a function then return adapted function. fuse and unfuse might be one of the most interesting adaptors, which provides "fusing".

Function Objects and Utilities

Function Objects provides useful functions which have been requested in Boost users list several times. For Utilities, infix might be interesting, which is ported from FC++.

Workarounds

Workarounds is set of workarounds. For now, only BOOST_EGG_CONST might be worth to read.
