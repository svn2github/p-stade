A Function Adaptor is a higher-order function that takes a "base" Function Object and returns an "adapted" Function Object. An adapted Function Object type meets the following unless otherwise specified:

  • It is a model of Major Function Object.
  • It is Stateless if and only if its "base" Function Object type is Stateless.
  • It supports Stateful static initialization using macros. 

int plusfun(int i, int j)
{
    return i + j;
};

static_< result_of_<T_curry2(std::plus<int>)> >::type const splus = {{}}; 
result_of_<T_curry2(int(*)(int, int))>::type const mplus = BOOST_EGG_CURRY2(&plusfun); 

void egg_example()
{
    result_of_<T_curry2(std::plus<int>)>::type dplus = curry2(std::plus<int>()); 

    BOOST_CHECK( splus(1)(2) == 3 );
    BOOST_CHECK( mplus(1)(2) == 3 );
    BOOST_CHECK( dplus(1)(2) == 3 );
}

static initialization for "stateless" functions

static initialization for "stateful" functions

dynamic initialization

In namespace scope, static initialization is highly recommended. For details about adapting "stateful" functions using macros, see Stateful static initialization.

Important

static_ can't work with an adapted function pointer, because it is not Stateless. See Stateful static initialization if you need an adapted function pointer. Also note that a Static Function Object type is guaranteed to work with static_.

ambiN

Description

The ambi family helps you to make Ambi Function Object.

Header
  • <boost/egg/ambi.hpp>
Model of
  • Static Function Object
Valid expressions

Valid expression

Semantics

X_ambi%%N<__Bytag = __ud>

A Major Function Object type

X_ambi%%N< __Bytag>()(__pfo)

An Ambi Function Object

o|X_ambi%%N< __Bytag>()(__pfo)(a1,...,aN)

__const(__pfo)(__fwd_arg_list([o]++a, __Bytag))

ambi%%N

X_ambi%%N<>()

Preconditions
  • N <= __MAX_PACK_ARITY-1.
  • If __Bytag is by_value, __typeof(aI) is Copy Constructible.
Example 

struct base_my_plus
{
    typedef int result_type;

    int operator()(int x, int y) const
    {
        return x + y;
    }
};

static_< result_of_<T_ambi1(base_my_plus)> >::type 
    const my_plus = {{}};

void egg_example()
{
    BOOST_CHECK( my_plus(1, 2) == 3 );
    BOOST_CHECK( ( 1|my_plus(2) ) == 3 );

    std::plus<int> std_plus;
    BOOST_CHECK( ambi1(std_plus)(1, 2) == 3 );
    BOOST_CHECK( ( 1|ambi1(std_plus)(2) ) == 3 );
}

Notice this is not ambi2 but ambi1.

compose

Description

compose is a Function Adaptor for function composition.

Header
  • <boost/egg/compose.hpp>
Model of
  • Static Function Object
Valid expressions

Valid expression

Semantics

X_compose<R0 = __ud, __Stg = __ud>

A Major Function Object type

X_compose<R0, __Stg>()(__pfo1, __pfo2)(a1,...,aN)

__const(__pfo1)(__const(__pfo2)(__fwd_arg_list(a, __Stg)))

compose

X_compose<>()

  • 0 <= N && N <= __MAX_PACK_ARITY.

    int r = (&increment ^compose^ &decrement ^compose^ &increment)(3);
    BOOST_CHECK( r == 4 );
}

curryN

Description

The curry family turn Function Objects into curried ones.

Header
  • <boost/egg/curry.hpp>
Model of
  • Static Function Object
Valid expressions

Valid expression

Semantics

curry%%N(__pfo)(a1)...(aN)

fix

Description

fix is Fixed point combinator, which enables you to write recursive functions on the fly.

Header
  • <boost/egg/fix.hpp>
Model of
  • Static Function Object
Valid expressions

Valid expression

Semantics

fix(__pfo)(a1)

__const(__pfo)(fix(__pfo))(__fwd_arg_list([a1], by_perfect))

fix2

Static compose(fix, curry2)

Note

fix(__pfo) that is passed to __pfo as the first argument represents __pfo itself. Also notice that __pfo must be curried in advance.

Example 

void egg_example()
{
    using bll::_1;
    using bll::_2;

    int r =
        fix2(
            bll::ret<int>(
                // \(f,a) -> a == 0 ? 1 : a * f(a-1)
                bll::if_then_else_return( _2 == 0,
                    1,
                    _2 * lazy(_1)(_2 - 1)
                )
            )
        ) (5);

    BOOST_CHECK(r == 5*4*3*2*1);
}

Invariants
  • An adapted Function Object doesn't support Stateful static initialization.

Boost.Phoenix is able to make a lambda expression without a "bind function". lazy turns a bindable Function Object into such one which can be used with Boost.Lambda.

Header
  • <boost/egg/lazy.hpp>
Model of
  • Static Function Object
Notation

Valid expressions

Valid expression

Semantics

X_lazy<Bind = __ud>

A Major Function Object type

{
    typedef int result_type;

    int operator()(int x, int y) const
    {
        return x + y;
    }
};

static_< result_of_<T_lazy(base_my_plus)> >::type

    using bll::_2;

}

memoize

Description

memoize stores the result of function for later reuse.

Header
  • <boost/egg/memoize.hpp>
Model of
  • Static Function Object
Notation
  • fixed is an unspecified unary Function Object which represents __pfo itself.
Valid expressions

Valid expression

Semantics

memoize(__pfo)

A Major Function Object

memoize(__pfo)(a1)

__fwd_arg_list([__const(__pfo)(fixed, __fwd_arg_list([a1], by_value))], by_value)

Preconditions
  • A Function Object returned from memoize(__pfo) must always be called with the same type arguments.

    }

nestN

Description

nestN is generalization of curryN and lazy, which enables you to write nested lambda expressions. N represents nesting level of lambda expression.

Header
  • <boost/egg/nest.hpp>
Model of
  • Static Function Object
Notation
  • bll_bind is an imaginary Function Object behaving as if it were boost::lambda::bind.
  • wrap_ref_ is an imaginary Function Object behaving as if it were boost::ref.
  • unwrap_ref_ is an imaginary Function Object which extracts a wrapped reference from boost::reference_wrapper.
  • bind_ is bll_bind if Bind is __ud, Bind() otherwise.
  • protect_ is boost::lambda::protect if Protect is __ud, Protect() otherwise.
  • LAZY(N) is:
    • If N == 0, X_identity<by_value>().
    • Else if N == 1, X_lazy<__typeof(bind_)>().
    • Otherwise, X_lazy<result_of_lazy<__typeof(bind_), result_of_lazy<__typeof(bind_),...N-1 times...>::type>::type>().
Valid expressions

Valid expression

Semantics

X_nest%%N<Bind = __ud>

A Major Function Object type

X_nest%%N<Bind>()(f)(a1,...,aK)

Polymorphic LAZY(N)(f)(__fwd_arg_list(a, by_value))

nest%%N(x)

X_nest%%N<>()(x)

X_ref%%N<Bind = __ud>

A Major Function Object type

X_ref%%N<Bind>()

Polymorphic compose(X_nest%%N<Bind>()(unwrap_ref_), wrap_ref_)

ref%%N

Static X_ref%%N<>()

XX_%%M%%_<Protect = __ud>

A Major Function Object type

XX_%%M%%_<Protect>()(p)

Polymorphic protect_(protect_(...M times...(p)...))

_%%M%%_

Static XX_%%M%%_<>()

using BOOST_EGG_NEST_NAMES;

using-declarations of nest%%N, ref%%N, _%%M%%_, and boost::lambda::_%%K.

Preconditions
  • 0 <= N && N <= BOOST_EGG_MAX_ARITY.
  • 0 <= M && M <= N-1.
Important

For capturing by-reference in nested lambda expressions, refN must be used instead of boost::ref.

Invariants
  • An adapted Function Object is not Stateless.
  • An adapted Function Object doesn't support Stateful static initialization.
Example 

int & second(int, int &j, int, int)
{
    return j;
}

void egg_example()
{
    using bll::_1;
    using bll::_2;
    std::plus<int> plus;
    std::minus<int> minus;

    int i6 = 6, i1 = 1, i3 = 3, i9 = 9;

// Lv: 0     1      2          3
    // \x -> (\y -> (\(z,w) -> foo(y,w,z,x))))
    BOOST_CHECK( nest3(foo)(_1_(_1), _2_(_2), _2_(_1), _0_(_1)) 
        (i3)(i6)(i1,i9) == foo(6,9,1,3) );

    // \x -> apply(\y -> minus(x,y), plus(x,3))
    BOOST_CHECK( nest1(apply)(nest2(minus)(_0_(_1), _1_(_1)), nest1(plus)(_0_(_1), 3)) 
        (i9) == minus(9, plus(9,3))  );

    int w = 7;
    // \x -> (\y -> (\z -> second(y,w,z,x))))
    BOOST_CHECK(
        boost::addressof(
            nest3(second)(_1_(_1), ref3(w), _2_(_1), _0_(_1)) 
                (i1)(i1)(i1)
        ) == boost::addressof(w) );
}

_M_ turns a placeholder into nested one.

By the definition, nest1 has the same semantics as lazy.

Captures w by-reference.

Stateful static initialization

description

For some reason, you need statically initialized "stateful" functions adapted by Function Adaptors. As you know, the initializer of static_ must be the empty double braces {{}}. How can we pass "states" to Function Adaptors? We need "ugly" macros.

Notation
  • adapt is one of the name of Function Adaptors.
  • ADAPT is the uppercase tokens of adapt.
  • BI is a "base" object type passed to adapt.
  • A_ is an "adapted" Function Object type from B_ by adapt.
  • wrap_ is an imaginary aggregate type which is defined as struct wrap_ { B1 b1;...BN bN; };.
Valid expressions

Valid expression

Semantics

BOOST_EGG_%%ADAPT%%_L init BOOST_EGG_%%ADAPT%%_R

A braced initializer of A_ object initialized by init_

BOOST_EGG_%%ADAPT(init)

BOOST_EGG_%%ADAPT%%_L init BOOST_EGG_%%ADAPT%%_R

Preconditions
  • There is a w_ such that wrap_ w_ = {init}; is a valid expression.
Invariants
  • A_ is POD if and only if any BI is POD.
  • A_ is Default Constructible if and only if any BI is Default Constructible.
  • A_ is Copy Assignable if and only if any BI is Copy Assignable.
  • A non-local A_ object with static storage duration is statically initialized if any BI is POD and any expression passed to the macro is a constant expression.
Example 

typedef result_of_<T_pipable(int(*)(int,int,int))>::type T_pfoo;
T_pfoo const pfoo = BOOST_EGG_PIPABLE(&foo);

typedef result_of_<T_curry2(T_pfoo)>::type T_cfoo;
T_cfoo const cfoo = BOOST_EGG_CURRY2_L BOOST_EGG_PIPABLE(&foo) BOOST_EGG_CURRY2_R; 

void egg_example()
{
    BOOST_CHECK( (1|pfoo(2,3)) == foo(1,2,3) );
    BOOST_CHECK( (1|cfoo(2)(3)) == foo(1,2,3) );
}

A macro invocation must be sandwiched by _L and _R form.
