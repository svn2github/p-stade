Adapting stateful functions

Macro naming conversion and semantics

For some reason, you need statically initialized "stateful" functions adapted by Function Adaptors. As you know, the initializer of static_ must be the empty double braces {{}}. How can we pass "states" to Function Adaptors? We need "ugly" macros.

Macro naming conversion and semantics

Notation
  • adapt is one of the name of Function Adaptors which supports static initialization.
  • ADAPT is the uppercase tokens of adapt.
  • b is a "base" Function Object passed to adapt.
  • a is an "adapted" Function Object from b by adapt.
Valid expressions

Valid expression

Semantics

BOOST_EGG_%%ADAPT%%_L init BOOST_EGG_%%ADAPT%%_R

A braced initializer of a

BOOST_EGG_%%ADAPT(init_)

BOOST_EGG_%%ADAPT%%_L init_ BOOST_EGG_%%ADAPT%%_R

BOOST_EGG_COMPOSE(f, g)

BOOST_EGG_COMPOSE_L f, g BOOST_EGG_COMPOSE_R

Preconditions
  • __typeof(b) b_ = init; is a valid expression. * init_` contains no commas and macro recursions.
Invariants
  • __typeof(a) is POD if and only if __typeof(b) is POD.
  • __typeof(a) is Default Constructible if and only if __typeof(b) is Default Constructible.
  • __typeof(a) is Copy Assignable if and only if __typeof(b) is Copy Assignable.
  • A non-local a with static storage duration is statically initialized if __typeof(b) is POD and the expression passed to the macro is a constant expression.
